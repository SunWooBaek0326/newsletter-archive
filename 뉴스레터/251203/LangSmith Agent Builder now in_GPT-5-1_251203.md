# LangSmith Agent Builder now in Public Beta

# 코드 없이 ‘에이전트 시대’를 연다: LangSmith Agent Builder 퍼블릭 베타 공개

[도입부 – Editor’s Note]  
생성형 AI를 실무에 제대로 쓰려면 ‘워크플로우 자동화’ 수준을 넘어, 스스로 판단하고 반복적으로 일을 해결하는 **에이전트(Agent)** 구축이 필수가 됐어요. 하지만 지금까지는 개발자조차 안정적인 에이전트를 만들기 쉽지 않았습니다.  
LangChain 팀이 이번에 공개한 **LangSmith Agent Builder 퍼블릭 베타**는 이 장벽을 사실상 무너뜨립니다. 대화만으로 프로덕션급 에이전트를 만들고, 조직 전체가 재사용·확장·관리할 수 있는 시대가 열린 셈이에요.

# 대화만으로 프로덕션 에이전트를 만든다는 것의 의미  
LangSmith Agent Builder는 기존 LLM 워크플로우 빌더와 완전히 다른 철학을 가집니다. 워크플로우 방식은 개발자가 단계별 로직을 일일이 설계해야 하고, 예외 케이스도 사전에 모두 정의해야 합니다.  
반면 에이전트는 상황에 따라 **스스로 계획을 세우고**, **필요한 도구를 반복 호출하며**, **피드백을 통해 지속적으로 개선**합니다.

> “이제 누구나 코드를 쓰지 않고 프로덕션 레벨 에이전트를 만들 수 있습니다.”

그동안 기업이 에이전트 기반 생산성 도구를 실험하기 어려웠던 가장 큰 이유 3가지 — 설계 난이도, 도구 선택의 복잡성, 거버넌스 문제 — 를 모두 해결합니다.

# 본론 – Deep Dive

## 1. 에이전트를 만드는 방식 자체가 달라졌다 💡  
Agent Builder는 마치 팀 동료에게 메시지 보내듯 대화하며 에이전트를 설계합니다.  
대화형 인터페이스는 다음을 자동 처리합니다.

- 프롬프트 작성(수백~수천 자 규모 포함)  
- 필요한 도구 탐색 및 선택  
- 작업 계획 생성  
- 복잡한 작업은 하위 에이전트(Subagent)에 자동 위임  

또한 ‘에이전트를 만드는 에이전트’라는 점이 핵심입니다. 전 세계 수백만 LangChain 개발자 사례에서 축적된 베스트 프랙티스를 그대로 활용해, 비개발자도 안정적인 에이전트를 만들 수 있게 합니다.

---

## 2. 퍼블릭 베타에서 무엇이 달라졌나 🚀  
이번 퍼블릭 베타에서는 기업 환경을 겨냥한 기능이 대거 탑재됐습니다.

- **BYOT(Bring Your Own Tools)**: MCP 서버로 내부 시스템·외부 API 연결  
- **워크스페이스 에이전트**: 조직 내 에이전트 공유, 템플릿 만들기, 1-click 복제  
- **멀티 모델 지원**: OpenAI ↔ Anthropic 선택  
- **API 호출**: 기존 앱·프로세스에 에이전트 직접 연결  
- **전용 UI 탭**: 관찰·추적·평가 기능을 몰라도 누구나 빠르게 빌드  

이로써 ‘개인은 쉽게 만들고, 조직은 안전하게 관리’하는 구조가 완성됐습니다.

---

## 3. 5분 만에 만드는 실전형 에이전트  
기업이 실제로 겪어온 두 가지 난제—정확한 툴 세팅, 속도·보안·자율성의 균형—을 동시에 해결합니다.

### 운영 방식은 ‘개발자’가 아닌 ‘매니저’에 가깝다  
원하는 작업을 자연어로 설명하고, 필요한 도구만 승인하면 됩니다. 나머지는 Agent Builder가 해결합니다.

![Prompt UI](https://blog.langchain.com/content/images/2025/12/ab-1.png)

### 에이전트 수정? 자연어로 말하면 끝  
프롬프트·설정 변경을 위해 다시 설계할 필요 없이 원하는 수정 지시만 하면 즉시 반영됩니다.

![Update prompt](https://blog.langchain.com/content/images/2025/12/ab-2.png)

### 조직 전체에서 복제·재사용·확장  
워크스페이스 내에서 에이전트를 템플릿처럼 복제해 팀별 버전을 만들 수 있습니다.  
MCP 기반 내부 도구 연결과 OAuth 인증도 지원해 비개발 팀도 자유롭게 활용할 수 있어요.

![Workspace Agents](https://blog.langchain.com/content/images/2025/12/ab-3.png)

## 4. 이미 나타난 활용 패턴 3가지 📊  

### 패턴 1) 역할 특화 리서치 에이전트  
- 세일즈: 매일 아침 고객사·참석자·과거 히스토리를 기반으로 ‘준비 리포트’ 자동 생성  
- 마케팅: 경쟁사 뉴스·발표·이벤트·제품 업데이트 자동 모니터링  
- 리크루팅: 기준에 맞는 후보 탐색 → 우선순위 매기기 → 초안 메시지 생성  

연속적 탐색 → 분석 → 통합이라는 ‘루프 기반 리서치’는 워크플로우 도구로는 구현하기 사실상 불가능한 영역입니다.

---

### 패턴 2) 정보의 흐름을 ‘프로젝트’로 바꿔주는 에이전트  
- Slack 메시지 → Linear 이슈 자동 생성  
- 티켓 패턴 분석 및 주간 리포트 제공  
- GitHub 변화 모니터링 → 문서 보완 추천  

업무 시스템 사이에 흩어진 정보를 묶어 ‘추적 가능한 단위’로 만드는 효과가 큽니다.

---

### 패턴 3) 시간을 벌어주는 커뮤니케이션 어시스턴트  
가장 단순하지만 효과가 큰 유형입니다.

- 이메일 자동 분류/우선순위/초안  
- 일정 집중시간 확보를 위한 캘린더 관리  
- Slack 활동 요약 및 액션 아이템 생성  

몇 분의 설정 작업이 매주 수 시간을 절약하게 해 줍니다.

# 결론 – 한국 개발자·기업에 주는 메시지 💬  
LangSmith Agent Builder는 ‘업무용 에이전트’의 진입 장벽을 근본적으로 낮추는 도구입니다.  
코드가 필요 없고, 조직 전체에서 재사용·확장·관리까지 가능한 에이전트 플랫폼은 지금까지 없었습니다.

한국 개발자 및 기업 관점에서 의미는 분명합니다.

- **PoC 단계에서 바로 실서비스 단계로 점프할 수 있는 인프라**  
- **비개발 부서까지 AI 자동화를 확산시키는 촉매**  
- **조직 내부 시스템과 안전하게 연결할 수 있는 거버넌스 완비**  
- **‘도메인 특화 에이전트’를 빠르게 반복 개선할 수 있는 구조**

AI 도입이 ‘실험’에서 ‘실전’으로 넘어가는 시점,  
에이전트를 직접 구축해보는 조직과 그렇지 않은 조직의 생산성 차이는 앞으로 더 벌어질 것입니다.  
LangSmith Agent Builder는 그 전환점의 중심에 있습니다.
